# @Method Decorator

The `@Method` decorator exposes public instance methods on a Stencil component. This page documents the test coverage, patterns, generated components, and implementation status.

## Decorator Properties

- returns: void | value | promise
- async: marks the method async (affects return type when returning a promise)
- args: none | one

## Test Cases

Test cases are organized by **return type** - from promise-returning methods to void methods. All `@Method` tests reside under <code>test/wdio/method</code>.

export const MethodGeneratedComponents = () => {

        index: i,

export const MethodTODOSection = () => {
  const items = overlay.items || [];
  const groupIn = (set) => items.filter(i => set.has(i.group));
  const promiseSet = new Set(['promise']);
  const valueSet = new Set(['value']);
  const voidSet = new Set(['void']);
  
  const caseNosFor = (arr) => {
    const s = new Set();
    for (const it of arr) {
      for (const n of (it.caseIds || [])) {
        s.add(n);
      }
    }
    return Array.from(s).sort((a, b) => a - b);
  };
  
  const promiseCases = caseNosFor(groupIn(promiseSet));
  const valueCases = caseNosFor(groupIn(valueSet));
  const voidCases = caseNosFor(groupIn(voidSet));
  
  const missingCases = [];
  if (promiseCases.length === 0) {
    missingCases.push({ id: 1, desc: 'Promise methods - expose public API and update DOM after resolution' });
  }
  if (valueCases.length === 0) {
    missingCases.push({ id: 2, desc: 'Value-returning methods - provide immediate results and update component state synchronously' });
  }
  if (voidCases.length === 0) {
    missingCases.push({ id: 3, desc: 'Void methods - trigger side effects without return values' });
  }
  
  if (missingCases.length === 0) {
    return null;
  }
  
  return (
    <div>
      {missingCases.map(({ id, desc }) => (
        <div key={id} style={{ marginBottom: '12px' }}>
          <strong>Test Case #{id} TODO</strong>: {desc}
        </div>
      ))}
    </div>
  );
};
