# Key Concepts

## Stencil.js Core Concepts

### Component System
- **Web Components**: Standards-based custom elements
- **TypeScript**: Strongly typed component development
- **JSX**: React-like templating syntax
- **Shadow DOM**: Encapsulated styling and behavior

### Testing Philosophy
- **Matrix-based Testing**: Systematic coverage across all combinations
- **Generated Components**: Automated test component creation
- **AI-Assisted**: GenAI integration for testing automation
- **Multi-layer Testing**: Unit, integration, e2e, and component tests

### Build System
- **Compiler**: TypeScript to optimized JavaScript
- **Runtime**: Minimal runtime for component hydration
- **Bundling**: Efficient code splitting and optimization
- **SSR**: Server-side rendering and hydration support

## Testing Framework Concepts

### Decorator Testing
- **@State**: State management testing via auto-generated components
- **Matrix Coverage**: All type/default combinations covered
- **Verification**: Automated coverage and structure validation

### AI Integration
- **Context Preservation**: Version-controlled AI memories
- **Team Consistency**: Shared AI understanding across developers
- **Automated Workflows**: AI-powered test generation and verification

### Coverage Strategy
- **Systematic**: Matrix-based approach ensures complete coverage
- **Automated**: Scripts handle generation and verification
- **Maintainable**: Clear structure and documentation
